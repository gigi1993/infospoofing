Source Address Validation


To prevent IP source address spoofing, network operators should do Source Address Validation. Source Address Validation refers to the practice of filtering packets with a non-verified source address by implementing ingress filtering according to BCP38 or BCP84, Best Common Practices to drop spoofed packets.

BCP38 (RFC2827) is the original document by the Internet Engineering Task Force. It describes a simple, effective, and straightforward method for using ingress traffic filtering to prohibit DoS attacks which use forged IP addresses to be propagated from 'behind' an Internet Service Provider's aggregation point.
However, there are cases when ingress filtering described in BCP38 may create problems, e.g., with multihoming. BCP84 (RFC3704) updates BCP38, and describes the current ingress filtering operational mechanisms, examines generic issues related to ingress filtering, and delves into the effects on multihoming in particular.

How to be removed from our list?

1) Contact us!

Send us a mail to infospoofing@tudelft.nl
When you communicate us your intention to implement anti-spoofing, we will change the status of your AS/IP from Spoofable to Remediating to show that you are taking remediating action! If in the next weeks we do not receive any spoofable test, the status will be updated to NAT-Blocking or UnSpoofable!

2) Download and do the test!

Download and run the testing application on the prefixes that presented evidences of spoofing. In this way we will have confirmation that you have deployed anti-spoofing filters!
