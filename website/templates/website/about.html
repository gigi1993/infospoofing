Who Are We?


We are a group of researchers of the Delft University of Technology (TU Delft), and we created this webpage to report the extent to which Autonomous Systems implement anti-spoofing. IP Spoofing is a critical vulnerability in Internet architecture, often exploited by attackers to launch massive amplification and reflection DDoS attacks. To mitigate this threat, Internet Service Providers (ISPs) can implement Source Address Validation according to BCP38 and BCP84, best common practices to implement anti-spoofing filters.
On this page, we report the measurements of compliance with Source Address Validation obtained with crowdsourcing campaigns in six countries, using CAIDA's Spoofer tool (see below). Our aim is to spread our results, with the hope that this information might increase awareness about which ISPs correctly implement Source Address Validation, and which instead allow IP spoofing on their network.

The Spoofer Project


Differently from other network proprieties, the compliance with Source Address Validation is extremely difficult to measure. So far, the most feasible way to test if a network is compliant is to try to send spoofed packets from that network, and evaluate if the operator properly filters its traffic.
To this aim, the Center for Applied Internet Data Analysis launched the Spoofer Project, providing volunteers with an application that tests the network on which it runs. The availability of information on the networks allowing IP Spoofing is crucial to mitigate the threat of DDoS attack, and the use of this application represents today the most reliable source.

HELP US! Download the test application!

Available for Windows, Mac and Linux systems, the application is very easy to install, and runs automatically without the need of special configurations. It takes less than 5 minutes, and it would help us in the fight against IP spoofing!
